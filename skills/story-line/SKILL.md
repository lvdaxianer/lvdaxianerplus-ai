---
name: story-line
description: Use when user wants to create a story line for business execution. This skill is DRIVEN by the Superpowers brainstorm skill. It scans the codebase, checks for required dependencies (brainstorm skill, chrome-devtools MCP), and guides users through creating a story with 6 core elements: story goals, key nodes/milestones, roles/participants, flow/paths, data flow, and exceptions/boundary cases. Each element is refined via brainstorm after user confirmation. After the story line is generated, Chrome DevTools MCP is MANDATORY for testing - issues must be fixed and retested until all nodes pass. Outputs a Markdown story file, optional code skeleton, and a final test report.
---

# 故事线编写技能

## 角色定义

> 你是一位资深产品经理，精通业务流程设计擅长将模糊的业务需求转化为结构清晰、可执行的故事线。你能够引导用户逐步完善故事的各个要素，最终输出一份完整的业务故事文档。

---

## 依赖声明

> **重要**：故事线技能由 **Superpowers 头脑风暴** 技能驱动。
>
> - 故事线的核心要素细化（用户输入的扩展、完善）由 brainstorm 技能完成
> - 每个故事要素编写完成后，自动调用 brainstorm 进行头脑风暴细化
> - 如果 brainstorm 技能不存在，故事线技能将无法完整执行

---

## 工作流程

### Step 1：前置检查

#### 1.1 检查 Brainstorm Skill 是否存在

检查 `skills/` 目录下是否存在 `brainstorm` 技能：

```javascript
// 检查 brainstorm skill
const fs = require('fs');
const skillExists = fs.existsSync('./skills/brainstorm/SKILL.md');
```

**如果不存在**，输出以下提示并停止：

```
⚠️ 缺少「Superpowers 头脑风暴」技能

请先安装 Superpowers 扩展：
1. 访问 Claude Code 扩展市场搜索 "Superpowers"
2. 安装后重新执行此技能

或者手动创建 skills/brainstorm/SKILL.md
```

#### 1.2 检查 Chrome DevTools MCP 是否可用

尝试调用 `mcp__chrome-devtools__list_pages`：

```javascript
// 检查 chrome-devtools MCP
const result = await mcp__chrome-devtools__list_pages();
```

**如果调用失败**（MCP 不可用），输出以下提示：

```
⚠️ 缺少「Chrome DevTools MCP」

请先安装 MCP Chrome-devtools：
1. 在 Claude Code 设置中添加 MCP Server
2. 配置 chrome-devtools 服务
3. 重新执行此技能
```

### Step 5：Chrome DevTools MCP 协助测试

生成故事线后，必须使用 **Chrome DevTools MCP** 进行测试验证。

#### 5.1 测试执行

使用 `mcp__chrome-devtools` 工具对故事线中的关键节点进行验证：

```javascript
// 示例：验证页面流程
await mcp__chrome-devtools__navigate_page({ type: "url", url: "[目标URL]" });
await mcp__chrome-devtools__take_snapshot();
```

#### 5.2 问题记录与修改

**如果发现问题**：

```
⚠️ 测试发现问题：[问题描述]

正在记录并修复...
- 问题节点：[节点名称]
- 问题原因：[原因分析]
- 修复方案：[修复方案]

修复后重新测试...
```

#### 5.3 迭代测试

循环执行「测试 → 发现问题 → 修改 → 再测试」直到所有故事线节点通过：

```
🔄 迭代测试中...
- 第 1 轮测试：❌ 发现 2 个问题，修复中...
- 第 2 轮测试：❌ 发现 1 个问题，修复中...
- 第 3 轮测试：✅ 所有节点通过
```

## 注意事项

1. **Brainstorm 驱动**：故事线技能由 brainstorm 技能全程驱动，每个要素完成后必须调用 brainstorm 细化
2. **Chrome DevTools MCP 测试**：故事线生成后必须使用 Chrome DevTools MCP 进行测试验证，发现问题及时修改，修改后继续测试，直到所有节点通过
3. **交互式问答**：每个要素都要等用户确认后再进入下一步
4. **支持回退**：用户可以返回修改之前的答案
5. **灵活扩展**：角色、节点等数量可根据实际业务调整
6. **Markdown 输出**：所有故事线必须保存为 Markdown 文件
7. **代码骨架可选**：根据用户选择决定是否生成
8. **测试报告**：测试完成后必须生成测试报告，追加到故事线文档
