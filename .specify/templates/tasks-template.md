# Tasks Template

> Template for generating implementation task lists

---

## Task Checklist Format

All tasks follow this strict format:

```text
- [ ] [TaskID] [P?] [Story?] Description with file path
```

- **TaskID**: Sequential number (T001, T002...)
- **[P]**: Parallelizable marker (optional)
- **[Story]**: User story label (US1, US2, US3...)
- **Description**: Clear action with exact file path

---

## Phase Structure

1. **Phase 1**: Setup (project initialization)
2. **Phase 2**: Foundational (blocking prerequisites)
3. **Phase 3+**: User Stories (US1, US2, US3...)
4. **Final Phase**: Polish & Cross-Cutting
