MCP 工具定义

三个核心工具的参数和返回值设计

🔧 generate_diagram

根据描述生成架构图

输入参数

{ "description": "画一个微服务架构，包含网关、订单服务、用户服务...", // 自然语言描述 "type": "deployment", // 可选: deployment | business | function "template": "microservice", // 可选: 使用预定义模板 "engine": "d2", // 可选: d2 | mermaid，默认 d2 "outputDir": "./diagrams" // 输出目录 }

返回结果

{ "success": true, "files": { "image": "./diagrams/arch-001.png", "code": "./diagrams/arch-001.d2" }, "preview": "base64_image_data...", // 可选: 图片预览 "message": "架构图生成成功" }

📋 list_templates

获取可用的架构图模板列表

输入参数

{ "type": "deployment" // 可选: 筛选特定类型的模板 }

返回结果

{ "templates": [ { "name": "microservice", "type": "deployment", "description": "微服务架构模板" }, { "name": "three-tier", "type": "deployment", "description": "三层架构模板" }, { "name": "c4-model", "type": "function", "description": "C4 模型模板" } ] }

📂 get_diagram

获取已保存的架构图

输入参数

{ "id": "arch-001", // 架构图 ID "format": "image" // 可选: image | code | both }

返回结果

{ "id": "arch-001", "image": "base64_data...", // 或文件路径 "code": "direction: right\n...", "metadata": { "type": "deployment", "created": "2026-04-20" } }

这三个工具定义覆盖了生成、查询、获取的核心流程。你觉得是否需要补充其他工具？
