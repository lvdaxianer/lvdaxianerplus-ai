技术方案对比

三种实现路径，各有优劣

A

独立 MCP 服务（推荐）

✓ 我的推荐

  • 职责清晰，独立演进
  • 不影响现有 mcp-http-gateway
  • 可单独部署和扩展
  • 依赖更少，更轻量

需新建项目结构

B

嵌入 mcp-http-gateway

作为现有项目的功能模块

  • 复用现有基础设施
  • 配置、日志、监控统一
  • 开发工作量较小

职责混杂，耦合度高

C

独立 MCP + 共享基础设施

新项目，复用通用工具库

  • 独立服务，职责清晰
  • 复用 types.ts 等通用定义
  • 配置风格保持一致

需维护共享库

推荐理由

架构图生成是独立能力域，与 HTTP 网关职责完全不同。 独立服务更易维护、测试、扩展，未来可单独迭代。
